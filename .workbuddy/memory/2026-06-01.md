## 2026-06-01

### 修复 sql.js WASM 文件路径问题

**问题**：Tauri DMG 打包后应用页面跑偏、功能不可用。根因是 `sql.js` 的 WASM 文件在 Next.js standalone 构建后找不到。

**根因分析**：
- Next.js/Turbopack 打包后的 chunk 中，`require.resolve('sql.js/package.json')` 被转换成数字模块 ID（如 59954），而非字符串路径
- `path.dirname(59954)` 报错：`TypeError: The "path" argument must be of type string. Received type number`

**解决方案** (`src/lib/db.ts`):
- 改用 `process.cwd()` 定位 WASM 文件路径
- `server.js` 中 `process.chdir(__dirname)` 确保 `process.cwd()` 指向 standalone 根目录
- 路径：`path.join(process.cwd(), 'node_modules', 'sql.js', 'dist')`

**修改文件**：
1. `src/lib/db.ts:35` - `wasmBaseDir` 改用 `process.cwd()` 替代 `require.resolve`
2. `src-tauri/tauri.conf.json:10` - `beforeBuildCommand` 简化，添加 JSON 数据文件复制

**验证结果**：
- Next.js standalone 服务器（端口 3030）全部 API 通过测试
- GET/POST/DELETE 操作正常
- JSON 到 SQLite 迁移成功（2本书）
- 数据库文件正确创建在 `data/library.db`
