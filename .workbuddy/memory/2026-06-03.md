### 修复 DMG 打包后样式丢失和按钮无反应的根因

**问题**：用户构建的 DMG 中页面无样式、按钮无反应，与上次修复的症状完全相同。

**根因**：`beforeBuildCommand` 第一行 `pnpm build` 实际调用 `scripts/build.sh`，该脚本第 9 行执行 `pnpm install --prefer-frozen-lockfile --prefer-offline`，在 Tauri 构建沙箱中因权限问题失败。由于 `set -Eeuo pipefail`，脚本立即中止，导致：
- `npx next build` 从未执行 → `.next/` 中无最新构建产物
- 后续 `cp` 命令全部被跳过 → `.next/static/`、sql.js WASM 等不会复制到 standalone
- Tauri 打包使用的是缓存的旧 `standalone.tar.gz`（缺少静态资源）

**修复**（`src-tauri/tauri.conf.json` 第 10 行）：
- `pnpm build` → `rm -rf .next && npx next build`：跳过 `scripts/build.sh`，直接执行 Next.js 构建
- 移除冗余的 `cp -r public .next/standalone/public`：standalone 自身已包含 public/

**完整 beforeBuildCommand 流程**：
1. `rm -rf .next && npx next build` — 清理构建
2. `cp server.js` — 自定义服务器（修复 308 重定向）
3. `cp .next/static` — 复制 CSS/JS 到 standalone
4. `cp sql.js WASM` — 复制数据库引擎
5. `cp JSON 数据文件` — 迁移数据
6. `tar czf` — 打包 standalone.tar.gz

**验证结果**：
- standalone 服务器 3035 端口测试：CSS 200（115KB）、JS 200、API 200
- tar.gz 内容验证：21MB，2091 个文件，CSS/JS/WASM/server.js/public/data 全部就位
