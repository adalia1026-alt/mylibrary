### 修复 DMG 打包后样式丢失和按钮无响应问题

**根因**：Next.js standalone 构建输出 `.next/standalone/.next/` 目录中**缺少三个关键目录**：

| 缺失项 | 影响 |
|---|---|
| `.next/static/` | CSS 和客户端 JS 全部 404，页面无样式、React 不水合 |
| `public/` | 静态资源（图片等）不可访问 |
| `node_modules/sql.js/dist/*.wasm` | 数据库 WASM 文件丢失，API 全崩 |

**解决方案**：在 `src-tauri/tauri.conf.json` 的 `beforeBuildCommand` 中增加三行复制命令：

```bash
# 1. 复制静态资源（CSS + 客户端 JS chunks）
cp -r .next/static .next/standalone/.next/static

# 2. 复制 public 目录
cp -r public .next/standalone/public

# 3. 复制 sql.js WASM 文件（Next.js 不会 trace pnpm 虚拟存储中的文件）
mkdir -p .next/standalone/node_modules/sql.js/dist
cp node_modules/sql.js/dist/*.wasm .next/standalone/node_modules/sql.js/dist/
```

**验证结果**（端口 3033）：
- 首页 HTML → HTTP 200，正确引用 CSS/JS
- CSS 文件 → HTTP 200，115KB（Tailwind 完整加载）
- JS chunk → HTTP 200，158KB（客户端水合正常）
- API /api/books → HTTP 200，书籍数据正常返回

**用户操作**：在 VS Code 终端运行 `pnpm tauri:build:mac` 重新打包即可。
