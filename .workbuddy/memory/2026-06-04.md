## 2026-06-04

### 修复 EPUB 导入三大问题

#### 1. 闪退（WRY NSOpenPanel crash）
- **根因**: macOS 26.3 上 WRY WebView 的 `NSOpenPanel::openPanel()` 触发 objc2 retain semantics panic
- **修复**: 
  - 添加 `tauri-plugin-dialog = "2"` (Rust)
  - 注册 `.plugin(tauri_plugin_dialog::init())` (lib.rs)
  - 创建 `src/lib/tauri-file-dialog.ts` 封装，Tauri 环境用 `__TAURI_INTERNALS__.invoke('plugin:dialog|open')` 替代 HTML `<input type="file">`
  - 添加自定义 Rust 命令 `read_file_bytes` 用于读取文件
  - 修改 `add-book-dialog.tsx` 的 EPUB/封面选择事件

#### 2. 封面提取失败
- **根因**: `epub2` 的 `metadata.cover` 返回对象 `{id: "xxx"}` 而非字符串，`manifest[coverId]` 查找失败
- **修复**: 在 `parse-epub/route.ts` 中对 cover 做 typeof 检查，提取 id 字段

#### 3. Nationality 🏁
- **根因**: `metadata.language` 可能为对象/数组，字符串处理失败导致 nationality 为空
- **修复**: 添加 object/array/string 多种格式的语言字段解析

#### 4. 数据持久化修复
- **根因**: 之前 `lib.rs` 每次启动删除 extract_dir 后重新解压，导致 `data/library.db` 被删
- **修复**: 改用 `tar xzf` 直接覆盖（不删除目录），保留不在 tar 中的运行时文件

### 修复封面图片无法显示

#### 5. trailingSlash 导致 308 死循环
- **根因**: `next.config.ts` 中 `trailingSlash: true` 导致 Next.js 16 standalone 对所有静态文件返回 308 重定向到同 URL → 无限循环
- **修复**: 改为 `trailingSlash: false`

#### 6. WebView 封面图片不渲染
- **根因**: macOS WKWebView（Tauri 底层）对同源图片加载有静默安全策略，不带 `crossorigin` 属性的 `<img>` 可能被拒绝而不报错
- **修复**:
  - `book-card.tsx` — `<img>` 添加 `crossOrigin="anonymous"` 和 `referrerPolicy="no-referrer"`
  - `server.js` — 添加文件级请求日志 `server-requests.log`（写入 Application Support 目录）
- **验证**: curl 直接请求返回 200 + image/jpeg + 126KB，服务器端完全正常
- **下一步**: 用户重新打开 DMG 应用后导入 EPUB 测试封面是否显示

#### 7. 封面还是不显示 —— add-book-dialog.tsx 被遗漏
- **文件确认**: 封面图片 `/uploads/1780542662528-...jpg` 存在且为有效 JPEG（通过 Read 工具可以清晰看到《博弈论》封面图）
- **服务器确认**: curl 直接请求返回 200 OK + `image/jpeg` + 126KB
- **根因**: 之前只在 `book-card.tsx` 加了 `crossOrigin`/`referrerPolicy`，但 **`add-book-dialog.tsx` 中有 2 个 `<img>` 标签都被遗漏了**
  - 搜索结果封面预览（searchResults.map 中的 <img>）
  - 详情页封面预览（details step 中的 <img>）
- **修复**:
  - `add-book-dialog.tsx` 两处 `<img>` 全部添加:
    - `crossOrigin="anonymous"`
    - `referrerPolicy="no-referrer"`
    - 相对路径 → 绝对路径：`typeof window !== 'undefined' && !src.startsWith('http') ? \`${window.location.origin}${src}\` : src`
  - 绝对路径转换是额外保险——确保 WKWebView 不会把 `/uploads/xxx.jpg` 解析到错误协议
